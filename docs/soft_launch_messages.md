# DeskLab Store — Soft Launch Messages

## Rule

Do not send spam. Do not pretend the store is already a complete e-commerce platform. The first goal is feedback and product-interest validation.

The correct framing is:

```text
I built a small static storefront to test whether desk setup accessories are interesting. It is not an automatic checkout yet: products are confirmed manually before any payment.
```

## Message 1 — Friends / classmates

```text
Ciao, sto testando una piccola idea: DeskLab Store, un mini catalogo di accessori per scrivania, studio e setup digitale.

Non è ancora un e-commerce automatico: è una fase di validazione. Mi servirebbe un feedback sincero: si capisce cosa vende? Ti sembra affidabile? C’è un prodotto che compreresti o che ti sembra utile?

Link:
https://antonmorosi06.github.io/desklab-store/

Se ti va, dimmi anche quale prodotto ti sembra più sensato e a che prezzo lo considereresti.
```

## Message 2 — Potential buyer

```text
Ciao, ho messo online DeskLab Store, un piccolo catalogo di accessori per scrivania e produttività.

Per ora funziona con richiesta disponibilità: scegli un prodotto, mandi una richiesta, poi confermo manualmente disponibilità, prezzo finale, tempi e condizioni prima di qualsiasi pagamento.

Link:
https://antonmorosi06.github.io/desklab-store/

Il prodotto principale che sto validando ora è lo stand laptop pieghevole. Ti interesserebbe una cosa del genere per studio/lavoro?
```

## Message 3 — Instagram story / short post

```text
Sto testando DeskLab Store: mini catalogo di accessori per scrivania, studio e produttività.

Prima fase: niente checkout automatico, solo richiesta disponibilità e conferma manuale prima del pagamento.

Mi serve feedback vero:
- si capisce?
- ti fideresti?
- quale prodotto compreresti?

Link:
https://antonmorosi06.github.io/desklab-store/
```

## Message 4 — More professional

```text
I have launched a small validation storefront called DeskLab Store.

It is a static e-commerce MVP for desk setup and productivity accessories, built with HTML/CSS/JavaScript and hosted on GitHub Pages. At this stage, it does not use automatic checkout: every product request is manually verified before payment.

Live site:
https://antonmorosi06.github.io/desklab-store/

I am collecting feedback on clarity, trust, product selection and first purchase interest.
```

## Message 5 — Follow-up after someone opens the site

```text
Grazie per averlo guardato. Ti faccio tre domande secche:

1. Si capisce subito cosa vende?
2. Ti sembra abbastanza affidabile da mandare una richiesta?
3. Quale prodotto ti sembra più utile o più debole?

Mi basta una risposta sincera, anche brutale.
```

## What to record after every message

For each person contacted, record:

- date
- channel
- person type
- message type
- whether they opened the link
- whether they gave feedback
- whether they liked a product
- whether they made a serious request
- notes
