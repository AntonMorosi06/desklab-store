# DeskLab Store — Validation Checklist

## Technical validation

- [ ] Site opens locally at `http://localhost:8000`
- [ ] Site opens on GitHub Pages
- [ ] `index.html` loads correctly
- [ ] `style.css` loads correctly
- [ ] `v04-premium.css` loads correctly
- [ ] `app.js` loads correctly
- [ ] `v04-premium.js` loads correctly
- [ ] `products.json` is valid JSON
- [ ] Product cards render correctly
- [ ] Images load correctly
- [ ] Mobile layout is usable
- [ ] Browser console has no critical errors
- [ ] All internal links work
- [ ] Legal draft pages are reachable if linked

## Product validation

- [ ] Each product has a clear target user
- [ ] Each product solves a simple desk/work/study problem
- [ ] Each product has a supplier cost target
- [ ] Each product has a margin goal
- [ ] Each product has a shipping estimate
- [ ] Each product has a risk level
- [ ] Product images are original or legally safe
- [ ] Product descriptions are honest
- [ ] No product promises unrealistic shipping
- [ ] No product claims unverified guarantees

## Business validation

- [ ] Manual order workflow is clear
- [ ] Payment is not automated too early
- [ ] Supplier availability can be checked before payment
- [ ] Customer communication is prepared
- [ ] Return/refund policy is reviewed before real sales
- [ ] Fiscal/tax position is reviewed before continuous selling
- [ ] Privacy notes are reviewed before collecting customer data

## Publication validation

- [ ] README is updated
- [ ] CHANGELOG is updated
- [ ] GitHub Pages link is visible
- [ ] Repository description is set
- [ ] Repository topics are set
- [ ] Screenshots are added or planned
- [ ] Release tag exists
- [ ] Limitations are documented
- [ ] Roadmap is documented
